# xiaofen.ai

## 项目介绍
xiaofen.ai 是一款面向增长型企业的 ToB AI SaaS 产品，核心目标是把优秀私域销冠的经验进行结构化沉淀、可复制执行与过程追踪，帮助团队从“依赖个人能力”升级为“可规模化的销售系统”。

我们将其定义为：**私域销冠复制智能体**。

## 产品定位：私域销冠复制智能体
xiaofen.ai 不替代销售，而是把销冠的关键动作拆解为可执行流程，通过 AI 在触达、对话、跟进、复盘环节提供辅助建议与自动化支持。

定位关键词：
- 面向私域成交场景
- 以“复制销冠方法论”为核心价值
- 强调团队化、流程化、可运营
- 以业务结果为导向（线索转化率、跟进效率、成交周期）

## 目标客户
- 以微信生态为主阵地的电商与品牌商家
- 有私域团队（3 人以上）并希望提升人效的企业
- 有成熟销冠个体，但缺少可复制 SOP 的组织
- 重视数据化运营、希望搭建长期私域资产的业务负责人

典型角色：
- 创始人 / 业务负责人
- 私域操盘手 / 运营总监
- 销售主管 / 组长

## 核心问题
当前私域业务常见痛点：
- **经验难复制**：销冠话术和判断依赖个人，无法快速复制到新人
- **执行不稳定**：跟进动作不一致，客户体验与转化波动大
- **管理不可视**：主管难实时判断“问题出在话术、节奏还是线索质量”
- **复盘低效率**：大量聊天记录难以形成可复用的策略资产

xiaofen.ai 聚焦解决：
1. 如何把销冠经验沉淀为“组织资产”
2. 如何让新人快速达到可成交水平
3. 如何让管理层用数据驱动持续优化

## MVP 功能
第一阶段 MVP 只覆盖最小闭环，确保“可上线、可使用、可验证价值”：

1. **销冠知识沉淀**
   - 上传历史聊天记录/话术文档
   - 自动抽取高频异议与有效应答
   - 形成可检索的话术知识库

2. **对话辅助建议**
   - 在沟通过程中提供下一句建议
   - 支持按场景调用（破冰、需求挖掘、异议处理、催付）
   - 提示风险表达与合规边界

3. **标准跟进流程**
   - 基于阶段定义跟进任务（首触达、二次唤醒、临门转化）
   - 提供待办看板与逾期提醒

4. **基础转化看板**
   - 线索量、触达率、回复率、成交率
   - 个人与团队维度对比
   - 常见丢单原因归因

## 技术栈说明
建议采用稳定、易扩展的主流技术栈：

- 前端：Next.js + TypeScript + Tailwind CSS + shadcn/ui
- 后端：NestJS（或 FastAPI）+ REST API
- 数据库：PostgreSQL
- 缓存与队列：Redis
- 向量检索：pgvector（MVP）
- AI 能力编排：LLM API + Prompt 模板 + RAG 检索
- 部署：Docker + 云主机/容器平台

## 本地启动方式
以下为推荐的本地开发流程（待代码实现后按实际命令更新）：

```bash
# 1) 克隆仓库
git clone <repo-url>
cd xiaofen-ai

# 2) 配置环境变量
cp .env.example .env.local

# 3) 启动依赖服务（PostgreSQL / Redis）
docker compose up -d

# 4) 安装依赖
# 前端
cd apps/web && pnpm install
# 后端
cd ../api && pnpm install

# 5) 启动开发服务
# 在 apps/web
pnpm dev
# 在 apps/api
pnpm start:dev
```
