# 大榕负责 AI 编辑部：落地操作方案

> 目标：把「大榕」从素材来源升级为可调度的“总编智能体”，并让现有 4 个 agent 形成稳定协作流。

## 0. 建议先做的结论（先拍板）

1. **从第 1 步开始**：先做素材索引，不直接写人格正文。
2. **保留旧的 4 个 workflow**：作为子流程；新增一个 `editorial_workflow` 统一编排。
3. **`darong-ai-test` 旧 Node 项目**：在新项目中写一条“历史系统引用”，不并入代码。
4. **`Randy-GPT-CODEX/docs` 产品文档**：先做“软链接式索引”（引用路径+摘要），不要复制全文。

## 1) 三阶段推进（推荐顺序）

### 阶段 A：建索引（1 天）

**产物**
- `docs/sources_index.md`

**你要做的事**
- 只登记来源：PPT/PDF/视频/历史项目文档。
- 每条仅保留：`来源路径 | 类型 | 可用于什么 | 敏感级别`。
- 不粘贴原文、不写推断。

**验收标准**
- 任意 agent 能在 2 分钟内找到“某观点可能在哪份资料里”。

---

### 阶段 B：搭骨架（1 天）

**产物**
- `darong_chief_editor.md`
- `workflows/editorial_workflow.md`
- 重写 `agents/01~04` 的结构模板（可先只改章节标题）

**你要做的事**
- 先定义结构，不填长内容：
  - 角色定位
  - 输入/输出
  - 决策权限
  - 上下游协作
  - 质量红线

**验收标准**
- 任何一篇内容都能按“总编→研究→结构→批判→主笔→总编复审”完整跑通一次。

---

### 阶段 C：写真人格（需授权读材料，2~3 天）

**产物**
- `darong_chief_editor.md`（完整版）
- `agents/04_chief_writer.md`（风格约束升级）

**你要做的事**
- 从授权资料提炼：
  - 大榕是谁（身份/定位）
  - 大榕怎么看（核心判断框架）
  - 大榕怎么说（语气、节奏、句式、禁忌）
- 形成“可执行提示词规则”，而非传记。

**验收标准**
- 给同一选题跑两次，输出风格稳定、观点边界一致。

## 2) 你的三个关键问题：建议定案

## Q1. 旧 4 个 workflow 保留还是删除？

**建议：保留。**
- 理由：它们本身是“场景化产线”（小红书、视频脚本、文章生成、选题讨论）。
- 做法：`editorial_workflow` 作为母流程，按任务类型调用子流程。

## Q2. `darong-ai-test` 要不要写进新项目？

**建议：写一条引用，不迁移代码。**
- 在 `docs/sources_index.md` 加“历史系统”分类。
- 标注：用途（历史 API 思路）、状态（只参考，不运行）、风险（含 `.env`）。

## Q3. `Randy-GPT-CODEX/docs` 产品蓝图要软链还是复制？

**建议：先软链式索引。**
- 只记路径 + 30~80 字摘要。
- 等到确认高频使用后，再抽取成“编辑部专用版本”文档。

---

## 3) 安全与边界（必须执行）

- 身份证、税务、执照等敏感文件：**不进 prompt，不进 agent 上下文**。
- 视频素材若无转写：不做内容细节推断，仅记录“可转写来源”。
- 每条外部素材都要记录最后一次确认日期，避免过时观点污染。
