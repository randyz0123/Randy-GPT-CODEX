## 1. 角色分工（先拍板）

### 1.1 Claude Code (CLI) 的职责（主设计）

1. 设计 AI 编辑部的总架构（角色、流程、信息流、质量门禁）。
2. 定义各 Agent 的职责边界与协作协议。
3. 产出“设计基线”文档，并维护版本演进。

### 1.2 Codex 的职责（执行+纠偏）

1. 把 Claude 的设计落地成可运行/可执行的文档与配置。
2. 做结构一致性检查（是否偏离设计基线）。
3. 做质量纠偏：
   - 流程缺步骤
   - 角色职责重叠
   - 输出标准不闭环
4. 给出可执行修订建议，并追踪整改状态。

### 1.3 你（Owner）的职责

1. 关键决策拍板（取舍、优先级、风险）。
2. 批准是否读取私有材料与范围边界。
3. 在设计冲突时做最终裁决。

## 2. 协作工作流（推荐）

### Step A：Claude 出“顶层设计稿”

建议 Claude 先交付以下 4 个核心件：
- `architecture_overview.md`（总架构）
- `agent_contracts.md`（各 Agent 合约：输入/输出/边界）
- `editorial_workflow.md`（主流程 + 子流程编排）
- `quality_gates.md`（质量门禁与验收标准）

### Step B：Codex 做“执行审计”

Codex 按固定清单审计 Claude 的设计稿：
1. **完整性**：是否覆盖“选题→研究→结构→批判→成稿→复审”。
2. **一致性**：不同文档中同一角色定义是否冲突。
3. **可执行性**：是否可以直接转成 prompt / SOP。
4. **可验证性**：每个阶段是否有验收标准。

输出审计件：
- `design_audit_report.md`（问题、等级、建议、责任人、截止时间）

### Step C：Claude 修订，Codex 复核

- Claude 按审计报告修订。
- Codex 做二次复核并标注“通过/需继续修订”。

### Step D：冻结基线，进入执行

基线冻结后，Codex 执行：
- 补齐骨架文件
- 对齐现有 Agent 文档
- 组织一次端到端空跑
